Webhooks can be created and managed by a Systems administrator or a user that is a member of a [role](https://www.contensis.com/help-and-docs/user-guides/governance/roles-and-permissions/assign-and-remove-role-permissions) with sufficient permissions.

The webhook resource supports the following actions.

Action

Description

Create

The ability to create a new webhook. Usually combined with the update action.

Update

The ability to save changes to a new or existing webhook.

Delete

Allow a member of a role to delete a webhook.

View

Allow members of a role to see the webhooks screen and activity.

Use custom templates

Determines whether a user can define a custom payload template or edit the default payload template.
